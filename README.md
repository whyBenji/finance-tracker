# Finance Tracker

This repo is set up as a professional starter project for learning how to build a real FastAPI application with:

- layered project structure
- SQLAlchemy models
- Alembic migrations
- CRUD API endpoints
- a reporting endpoint for monthly spending by category

## What This Version Does

The first version focuses on one core workflow:

- create a transaction
- list transactions
- update or delete a transaction
- view a monthly summary grouped by category

The data model includes:

- `date`
- `description`
- `category`
- `amount`
- `is_subscription`

It also adds `id`, `created_at`, and `updated_at`, which are standard in professional projects.

## Local Setup

1. Install dependencies:

```bash
uv sync
```

2. Create a local environment file:

```bash
cp .env.example .env
```

3. Apply the database migration:

```bash
uv run alembic upgrade head
```

4. Start the API:

```bash
uv run uvicorn app.main:app --reload
```

5. Open the docs:

```text
http://127.0.0.1:8000/docs
```

6. Run tests:

```bash
uv run pytest
```

## Migration Workflow

When your schema changes, the normal workflow is:

1. Update SQLAlchemy models.
2. Create a migration.
3. Review the migration file carefully.
4. Apply the migration.

Commands:

```bash
uv run alembic revision --autogenerate -m "describe change"
uv run alembic upgrade head
```

Do not treat autogenerated migrations as correct by default. Read them before applying them.

## Example Endpoints

- `POST /api/transactions`
- `GET /api/transactions`
- `GET /api/transactions/{transaction_id}`
- `PATCH /api/transactions/{transaction_id}`
- `DELETE /api/transactions/{transaction_id}`
- `GET /api/reports/monthly-summary?year=2026&month=5`

## Example Request

```bash
curl -X POST "http://127.0.0.1:8000/api/transactions" \
  -H "Content-Type: application/json" \
  -d '{
    "date": "2026-05-01",
    "description": "Groceries",
    "category": "food",
    "amount": "84.25",
    "is_subscription": false
  }'
```

## What To Build Next

After this starter is working, the next professional steps are:

1. Add `users` and authentication.
2. Attach every transaction to a `user_id`.
3. Add `categories` and `budgets`.
4. Add tests for services and API routes.
5. Switch local development to PostgreSQL.
6. Deploy with Docker and a managed database.
